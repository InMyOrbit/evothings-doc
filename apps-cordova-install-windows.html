Installing Cordova on Windows

This document describes how to install Apache Cordova and the Android SDK on a Windows machine.

Install Cordova

Follow these steps to install Cordova:

  1. Install Node.js from: http://nodejs.org
  2. Open a terminal window.
  3. Test the installation by typing:
    node --version
    If you get the version number is response, Node.js is installed and working!
  4. You also need the git version control system, which is used by Cordova behind-the-scenes. If you don't have git installed, download and install from: http://git-scm.com
  5. Test git by typing:
    git --version
    If you see the version number, git is installed!
  6. Install Cordova by typing:
    sudo npm install -g cordova
  7. Test the Cordova install by typing:
    cordova --version
    If you see the version number, you have succeeded!

Install Android SDK

To develop Android apps, install the Android SDK as follows:

  1. The Android SDK needs the Java SDK to be installed (min version 1.6). To check if you have Java already, type this on the command line:
    javac -version
    If you see a version number at or above 1.6 you should be good to continue. Otherwise, get Java for OS X.
  2. Download and install the Android Developer Tools (ADT).
  3. Make sure you have the path to the Android SDK tools added to the file .bash_profile, which is in your home directory. The path is needed for Cordova to find the Android build tools. Check if the build tools can be found by typing:
    adb version
android
    The first command should print a version number, the second command should start the android tools application (quit this if it launches). If these commands do not work, you need to add the path to the Android SDK by editing the .bash_profile file.
  4. If you need to edit .bash_profile, type the following in the terminal window (the cd command takes you to your home directory, in case you are unsure about where you are located in the file tree).

    5. cd
nano .bash_profile
    Now the nano editor opens (but you can use any editor you prefer). Enter the following, where you substitute "/Users/username/android" with the actual location where the Android SDK is installed (you can open the Finder or a second terminal window to search for the SDK directory):
    export PATH=/Users/username/android/sdk/platform-tools:/Users/username/android/sdk/tools:$PATH
    Save and exit by typing CTRL+O (the letter Oh) and CTRL+X.
  5. Now try the commands adb version and android again.
  6. If you get stuck, consult the Cordova documentation for Android.

Android SDK should now be ready to be used by the Cordova build tools. Follow the instructions below to see this in action!
