API Overview

The Evothings Viewer app ships with selected Cordova plugins, which are listed below.

Evothings also provides a number of JavaScript libraries, some of which are built on top of the Cordova plugins, some which are generic.

Below you find API documentation for plugins and libraries.

Evothings JavaScript libraries

The Evothings JavaScript libraries provide useful functionality for IoT enabled mobile apps.

These libraries are included using a script tag in the HTML code, or asynchronously using the evothings.loadScript() function.

Here is an overview of the libraries:

  • Evothings.js - evothings/evothings.js contains functions for asynchronous loading of JavaScript files and platform checking functions.
  • EasyBLE - evothings/easyble/easyble.js contains high-level functions for accessing BLE devices. This library is written on top of the cordova-ble plugin, and asynchronously includes the Util library.
  • Util - evothings/util/util.js contains functions for parsing numbers encoded as binary data. This library is used by the EasyBLE library.
  • TISensorTag - evothings/tisensortag/tisensortag.js is a high-level library for the TI Bluetooth Smart SensorTag. This library asynchronously includes the EasyBLE library.
  • Microcontrollers - Libraries are provided for a different microcontrollers, such as Arduino, Nordic Semiconductor, and RFduino. These are basic libraries that support connecting to a device and writing data.

When using a library that asynchronously includes another library using evothings.loadScript() it is good practice to always wait for all libraries to have loaded before calling any library functions. This is typically done in the Cordova 'deviceready' event handler. Here is an example:

function initialise()
{
    // All async scripts are loaded, the app is
    // ready to be initialised.
}

document.addEventListener(
    'deviceready',
    function() { evothings.scriptsLoaded(initialise) })

Browse the JSDoc generated API documentation for the Evothings JavaScript libraries to learn more.

The source code for the libraries is found on GitHub: https://github.com/evothings/evothings-examples/tree/master/resources/libs/evothings

Some of the JavaScript libraries are written to communicate with Arduino sketches. The Arduino source code is found here: https://github.com/evothings/evothings-examples/tree/master/resources/arduino

Please note that the example app code in the evothings-examples repository on GitHub does not contain any library files in the application folders. You must run a build script which copies library files in the resources folder to each example. This is done to avoid code duplication in the GitHub repository and make maintenance easier. Note that in the download package of Evothings Studio the examples are already built and include all required libraries. You only need to build examples if you are working with the files in the GitHub repository.
